**HDPRF** is a Pseudo-Random Number Generator (PRNG) framework designed for statistical complexity research. It utilizes multi-layer polynomial mixing, high-dimensional hypercube state traversal, and adaptive fallback mechanisms to produce statistically robust random byte streams.

This repository contains the official implementation of the framework. Core computational operations are accelerated using Just-In-Time (JIT) compilation to support parallel throughput and ensure reproducibility for the experimental results evaluated in our research.

## Key Algorithmic Features

- **Multi-Layer Polynomial Mixing:** Computes randomness across dynamic iterations ($F_1 \to F_N$), resolving polynomial coefficients adaptively.
- **State Space Traversal Logic:** Utilizes a `global_offset` mechanism to maintain continuous state space traversal across parallelized chunks, mitigating phase offset collisions.
- **External Entropy Injection:** Supports initialization via OS-level CSPRNGs (`os.urandom`) or arbitrary external binary seeds (e.g., thermal noise data).
- **JIT-Optimized Kernels:** Computationally intensive loops are parallelized using Numba (`@jit(nopython=True, parallel=True)`) to bypass the Python GIL and minimize dynamic memory allocation.

## Installation

It is recommended to run this framework within a virtual environment.

```bash
# 1. Clone the repository
git clone https://github.com/YourUsername/HDPRF.git
cd HDPRF

# 2. Create and activate a virtual environment
python -m venv venv
source venv/bin/activate  # On Windows use: venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt
```

## Quick Start & Reproducibility

The framework is configuration-driven. You can generate a random byte stream using the default parameters provided in the repository.

### Standard Execution

```bash
python src/main.py --config configs/default_config.json
```

### Performance Profiling

To analyze execution times and verify JIT acceleration, append the `--profile` flag:

```bash
python src/main.py --config configs/default_config.json --profile
```

### Configuration Adjustments

To reproduce specific experimental setups from our paper, modify `configs/default_config.json`:

- `dimension`: Adjusts the underlying hypercube dimensions.
- `iteration_count`: Sets the depth of the polynomial layers ($F_N$).
- `external_entropy_source`: Path to a custom noise seed (leave empty to default to the OS CSPRNG).
